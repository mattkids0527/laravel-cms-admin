### Bug 修復記錄

**Tailwind JIT purge 問題（關鍵）**
- 專案剛從 CDN 遷移 Vite，M006 新增的所有響應式 class（`md:block`、`md:hidden`、`md:flex-row` 等）均未被掃描進 build
- 解法：所有響應式邏輯一律移入 `admin.blade.php` 的 `<style>` block，用自訂 CSS class 名稱，完全不依賴 Tailwind utility 的 responsive prefix
- 此原則適用後續所有新增響應式或 JS-toggle 行為

**Alpine scope 問題**
- backdrop、aside、漢堡按鈕、主題切換各自有獨立 `x-data`，跨 scope `$store` 響應在 Livewire 3 載入 Alpine 的環境中失效
- 解法：`x-data` 移到最外層 `<div class="layout-root" x-data>`，所有子元素共用同一 Alpine scope
- 保留有本地狀態的元素（nav link tooltip：`x-data="{ tooltipVisible: false }"`）

### 表格操作欄 UX 迭代

**問題**：手機螢幕寬度縮小後，操作欄（編輯/刪除）被壓縮看不到

**迭代 1 — ⋮ 下拉選單**：操作欄改為三點按鈕展開 dropdown，但點擊目標太小，體驗不佳

**迭代 2 — 點列觸發**：改為點擊整列觸發彈出，底部 bottom-sheet modal 顯示操作，解決點擊目標問題

**迭代 3 — 最終設計（已實作）**：
- 操作欄 `<th>` 加 `table-action-th`、`<td>` 加 `table-action-td`
- 手機下：`table-action-th` → `display: none`；`table-action-td` → `padding: 0 !important; width: 0`（不用 `display: none`，避免 `position: fixed` 子元素失效）
- `table-desktop-actions` 在手機隱藏
- 點整列觸發 bottom-sheet modal：`x-data="{ open: false }"` 在 `<tr>` 上，`@click="if(window.innerWidth < 768) open = true"`
- Modal 結構：固定底部（`.row-action-overlay`）+ 卡片（`.row-action-card`）
- 卡片按鈕：全部 border 樣式、置中文字，無「取消」按鈕（點遮罩關閉）
- `@click.stop` 用於阻止 overlay/card 點擊事件冒泡到 `<tr>` 重複開啟

### Bottom-sheet 動畫

- 採 Alpine `x-transition:enter-start` / `x-transition:leave-end` 配合純 CSS transition，不依賴 Tailwind 編譯
- 開啟：遮罩淡入（`opacity: 0 → 1`）+ 卡片從下方滑入（`translateY(4rem) → 0`）
- 關閉：反向播放，動畫結束後 Alpine 才設 `display: none`
- `.row-overlay-hidden` 為 Alpine transition helper class，定義於 `admin.blade.php` `<style>` block

## 風險與注意事項

- 所有響應式樣式集中在 `admin.blade.php` `<style>` block，未來若 block 變大需考慮拆出獨立 CSS 檔
- `table-action-td` 使用 `width: 0` 而非 `display: none`，是為了讓 `position: fixed` 的 bottom-sheet 能正常顯示；若未來改用 portal/teleport 方案可移除此限制
- 點列開啟 modal 的 `@click` 判斷用 `window.innerWidth < 768` 而非 CSS，若日後需要支援摺疊式視窗或平板可能需調整閾值
