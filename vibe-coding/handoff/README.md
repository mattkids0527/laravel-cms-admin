# README.md

## 這份文件做什麼

- 用來說明 `vibe-coding/handoff/` 目錄中的交接文件怎麼建立與延續
- 這份文件幫助不同時間點接手的人快速理解目前進度與下一步
- 若交接內容開始混亂，應優先回到這份文件確認命名與內容規則

## Handoff 誰負責整理

- handoff 以 AI 協助整理為主
- 使用者不需要自己手寫完整交接內容，只需要決定是否要記錄
- AI 應協助整理目前進度、下一步與風險，並寫入 handoff 文件

## Handoff 觸發時機

- 完成一個明確任務後
- 每段工作階段完成後，若已完成或暫停 1 個以上 milestone 或 task
- 準備切換主題前
- 準備暫停或結束目前工作前
- 發現仍有未完成風險時
- 準備交由下一次對話或下一位協作者接續時

## Handoff 位置解析

- 若 `AGENTS.md` 已明確定義 handoff 路徑，優先使用該路徑
- 若 `AGENTS.md` 未明確定義 handoff 路徑，第一次記錄進度前應先找 `vibe-coding/handoff/`
- 不可在檢查明確路徑與 `vibe-coding/handoff/` 前，自行決定其他 handoff 資料夾
- 若靠 fallback 找到 handoff 位置，應建議把該路徑補回 `AGENTS.md`

## Handoff 提示方式

- AI 可主動詢問：
  - 是否要我整理目前 handoff？
  - 這一段已完成，是否要記錄成 handoff？
  - 目前已有可交接內容，是否要我寫入 handoff？
  - 這段工作階段已完成，是否要我整理交接記錄？
