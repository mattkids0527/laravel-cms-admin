# M002 — 帳號與角色管理系統

## 目標

建立 CMS 後台的帳號管理與角色管理功能，包含帳號 CRUD、角色 CRUD、帳號狀態管理、登入驗證，以及帳號與角色的多對多關聯。

## 設計確認

- 一個帳號可擁有多個角色（多對多，`role_user` 中介表）
- 登入方式：Email + 密碼
- 帳號狀態：待審核、啟用、停用、已刪除（軟刪除）
- 權限系統：完全自建，不使用 Spatie
- 前端：Blade + Livewire
