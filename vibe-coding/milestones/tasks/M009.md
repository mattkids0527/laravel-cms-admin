# M009 — 後端模組化架構遷移（nwidart/laravel-modules）

## 說明

將現有 `app/` 下的業務邏輯依功能職責拆分為獨立的 Laravel Module。
套件採用 `nwidart/laravel-modules`，各模組擁有自己的 Controllers、Livewire、Models、Routes、Views、Seeders。

## 模組規劃

| 模組 | 職責 | 包含內容 |
|------|------|----------|
| `Auth` | 認證（登入 / 登出） | AuthController, Login Livewire, guest layout |
| `Account` | 帳號與角色管理 | User model, Role model, Users/Roles Livewire, 對應 Seeders |
| `Menu` | 後台選單與選單權限 | AdminMenu model, AdminMenuService, MenuPermissionIndex Livewire, CheckAdminMenuPermission middleware, AdminMenuSeeder |
| `Settings` | 外觀偏好設定 | AppearanceSettings Livewire |
| `Dashboard` | 儀表板 | Dashboard Livewire |

## 不遷移範圍

- `resources/views/components/` — 共用 UI 元件，保留原位
- `database/migrations/` — 已執行的 migration 保留原位，新功能才放入模組
- Laravel 核心 Middleware（Authenticate, VerifyCsrfToken 等）— 保留在 `app/Http/Middleware/`
- `app/Providers/` — 保留原有 Providers，模組各自有 ServiceProvider

## 注意事項

- 每個 task 遷移完後應立即確認對應功能正常，不要等到 T09 才驗證

- 路由名稱須與原 `admin.*` 路由保持完全一致，避免 view 中 `route()` 呼叫失敗
