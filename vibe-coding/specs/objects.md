# objects.md

## 這份文件做什麼

- 用來定義此產品專案中的物件
- 物件是會被修改、被傳遞，或本身持有狀態的內容
- 每個物件至少應說明定義與屬性，若物件有狀態，也應一併記錄

## 範例

### 任務

- 定義：系統中可被建立、指派、追蹤與完成的工作單位
- 屬性：
  - 標題
  - 描述
  - 負責人
  - 到期日
- 狀態：
  - 待處理
  - 進行中
  - 已完成

### 使用者資料

- 定義：系統中用來表示使用者身分與基本資訊的物件
- 屬性：
  - 名稱
  - 電子郵件
  - 角色

### 申請單

- 定義：系統中用來表示一筆待審核、可被處理與結案的申請資料
- 屬性：
  - 申請編號
  - 申請人
  - 申請內容
  - 審核意見
- 狀態：
  - 草稿
  - 已送出
  - 審核中
  - 已核准
  - 已退回
  - 已取消

## 正式資料

### 帳號

- 定義：系統中可透過 Email 登入並執行後台操作的使用者身分
- 屬性：
  - 姓名
  - Email（唯一，作為登入憑證）
  - 密碼（雜湊儲存）
  - 角色列表（多對多，可擁有多個角色）
  - 狀態
  - 建立時間
  - 最後登入時間
- 狀態：
  - 待審核
  - 啟用
  - 停用
  - 已刪除（軟刪除，資料保留）

### 角色

- 定義：定義一組後台功能與選單授權的身分標籤，可同時指派給多個帳號
- 屬性：
  - 名稱（唯一）
  - 描述
  - 選單權限列表（多對多）
  - 是否為系統保護角色（用於保護超級管理員不被刪除）
