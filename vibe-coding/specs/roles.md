# roles.md

## 這份文件做什麼

- 用來定義此產品專案中的角色
- 角色是互動中的位置，用來描述誰發起行為、誰接收結果
- 這份文件不處理物件、流程或規則本身

## 範例

### 使用者

- 定義：使用系統功能並觸發主要操作的一方

### 管理者

- 定義：負責管理系統設定、審核或高權限操作的一方

### 系統

- 定義：接收輸入、執行處理並回傳結果的一方

## 空白模板

```md
### 角色名稱

- 定義：
```

## 正式資料

### 超級管理員

- 定義：可管理所有帳號、角色、系統設定與選單權限，不受任何功能限制

### 一般管理員

- 定義：可操作被授權的功能模組，無法管理角色定義與系統層級設定

### 編輯人員

- 定義：可新增、編輯被授權的內容，無法管理帳號或角色

### 檢視者

- 定義：只能查看資料，不可執行新增、編輯、刪除等操作
